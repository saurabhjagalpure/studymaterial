Install &
Launch

Node.js 18+ required. Runs on macOS, Linux, and Windows (WSL). Available via CLI, VS Code, JetBrains, and claude.ai/code.

npm install -g @anthropic-ai/claude-code
2
Navigate to your project & launch
cd my-project && claude

Run /init to generate CLAUDE.md

📄 Layer 1

CLAUDE.md — Project Memory

Loaded at every session start. Claude's permanent context about your project — stack, conventions, architecture, and forbidden patterns.

Loaded every session
Commit to git
Hierarchical (per-dir)
Keep under 200 lines
What to include:
→ Tech stack — frameworks, databases, key libraries
→ Coding conventions — patterns to always follow
→ Architecture — key directories and what lives there
→ Never-do rules — explicit prohibitions
→ Test command — how to verify changes
⚠ The 150-instruction limit

LLMs reliably follow ~150 instructions max. Claude Code's system prompt uses ~50. You get ~100 slots. More instructions = less reliable attention to critical rules. Keep it focused.

CLAUDE.md — example
# Acme E-commerce Platform ## Tech Stack Next.js 14 (App Router), FastAPI, PostgreSQL Turborepo monorepo, Railway deployment ## Coding Conventions - TypeScript strict mode always on - Functional components only - Zod for all validation schemas - `{ data, error }` API response shape ## Architecture /lib/db/repos/ — repository pattern /lib/auth.ts — JWT auth utilities /app/api/ — API route handlers ## Never Do - Never use any (cast via unknown) - Never console.log in production - No class components ## Testing Run: `npm run type-check && npm test`

⚡ Layer 2

Skills &
Commands

Markdown files that package reusable workflows. Invoked as /slash commands or auto-triggered when Claude detects relevance.

/code-review
/deploy
/research
/test-fix
/document
3 invocation modes
① /slash-command — user triggers manually
② Auto-trigger — Claude detects relevance
③ context: fork — runs in sub-agent
.claude/skills/code-review/SKILL.md
--- name: code-review description: Review code changes for quality and bugs. Use when asked to review a PR or commit. allowed-tools: Read, Grep, Glob, Bash(git:*) context: fork --- Review: $ARGUMENTS Check for: logic errors, error handling, TypeScript types, security, performance. Return: SUMMARY + ISSUES (severity) + SUGGESTIONS
Key frontmatter fields
name
The /slash-command name for invocation
description
Used by Claude for auto-invocation matching
allowed-tools
Whitelist: Read, Write, Bash(cmd:*), Grep…
context: fork
Runs in isolated sub-agent, keeps main clean
disable-model-invocation
true = user-only trigger (for deploy scripts)

Agents

Pre-configured AI specialists — each with their own system prompt, tools, model, and memory. The cure for context pollution.

🔍

researcher

Deep web research on any topic. Runs in background while you keep coding.

WebSearch WebFetch Read
✅

code-reviewer

Reviews diffs for bugs, types, security. Read-only access by design.

Read Grep Bash(git:*)
🗄️

db-explorer

Read-only database queries. Hook enforces no writes at the code level.

Bash(psql:*) Read
memory: project — agent builds up institutional knowledge across sessions. First thing it does each session: read its own MEMORY.md.

🪝 Layer 5

Hooks

Deterministic code at lifecycle events. Unlike instructions to Claude, hooks always run exactly the same way. No hallucinations, no variation.

PreToolUse

Fires before any tool. Can block dangerous operations.

Can block
PostToolUse

After a tool completes. Run lint, notify, log.

Read-only
PreAgentTurn

Before each turn. Inject date, env vars, context.

Read-only
PostAgentTurn

After each turn. Desktop notify, Slack alert.

Read-only
Stop

Task complete. Trigger CI, send notifications.

Read-only
~/.claude/settings.json
{ "hooks": { "PostToolUse": [{ "matcher": "Write", "hooks": [{ "type": "command", "command": "npm run lint --fix" }] }], "Stop": [{ "type": "command", "command": "./scripts/notify.sh" }] } }
